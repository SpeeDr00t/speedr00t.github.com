An attacker can exploit these issues through a browser. To exploit a cross-site scripting issue, the attacker must entice an unsuspecting victim to follow a malicious URI.

The following example URIs are available:

http://www.example.com/news/preview.php?id=SQL-INJECTION
http://www.example.com/news/preview.php?p=SQL-INJECTION
http://www.example.com/news/admin.php?act=news&amp;orderType=ASC/DESC&amp;search=&amp;orderBy=SQL-INJECTION
http://www.example.com/news/preview.php?id=`14&amp;p=`&amp;search=CROSS SITE SCRIPTING
http://www.example.com/news/admin.php?act=news&amp;orderType=`CROSS SITE SCRIPTING
http://www.example.com/news/admin.php?act=news&amp;orderType=CROSS SITE SCRIPTING&amp;search=&amp;orderBy=CROSS SITE SCRIPTING
http://www.example.com/news/preview.php?act=news&amp;orderType=CROSS SITE SCRIPTING
