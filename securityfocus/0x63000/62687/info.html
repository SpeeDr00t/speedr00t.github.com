An attacker can exploit this issue by enticing an unsuspecting victim to following a malicious URI.

The following example URI is available:

http://www.example.com/jforum/jforum.page?module=user&amp;action=validateLogin&amp;returnPath=http://www.example1.com&amp;username=user&amp;password=pass&amp;redirect=&amp;login=Login
